# SynthGen v2 — next iteration plan

v1 passed the adoption gate (8/9 ledger arms) but the exam surfaced four
defects. v2 fixes what the evidence indicts, nothing else. Every change bumps
`recipe.VERSION` → new `syn:v2/*` namespace → re-freeze → full 9-arm backtest
re-run; v1 results stay comparable under their own keys.

## Fixes (evidence → change → acceptance)

1. **Shallow-target tilt** (depth4 arm: +0.31% overall; wins on saturated
   cell-rules, n<2000, cat lookups; reality-shaped slices agree with ledger).
   - Deepen the prior: `interaction_depth` {1:.10, 2:.30, 3:.35, 4:.15, 5:.10};
     raise mean in-degree; keep saturated share but exclude saturated sets from
     depth/lr arm judgments (they reward low capacity by design).
   - Freeze composition: stratify screen n to the corpus shape inside
     [600, 8000] instead of greedy small-n fill (cap n<2000 share at ~35%).
   - Accept: depth4 arm flips to loss (or not-win) while depth8/lr03 stay
     losses.

2. **Entity-effect categoricals** (realism CHECK: CatBoost>LGBM winrate 0.45
   on high-card cats vs 0.62 numeric — real data shows the opposite).
   - New cat mechanism alongside the discretized-view one: category = latent
     entity; per-level random effect enters the SCM as a parent node; Zipf-ish
     level frequencies; cardinality up to 64; a few test-only rare levels.
     ~40% of cat columns draw the entity mechanism.
   - Accept: high-card-cat winrate CHECK flips to PASS (CatBoost's moat
     reproduces); backtest still ≥7/9.

3. **Earned canaries** (v1 canaries were canaries by construction; the
   cat_cross ones turned out to be genuinely hard car-analogs; screen has zero
   cat-bearing verified canaries).
   - Freeze-time check: fit default ChimeraBoost on every saturated candidate;
     `canary=True` in meta only if excess Brier ≤ 0.02 / excess RMSE ratio
     ≤ 1.1. Require ≥3 cat-bearing canaries in the screen; keep the hard
     cat_cross sets as the car-analog "+" slice (they earned that job).
   - Accept: catcombo canary slice non-empty and non-positive.

4. **Watch items** (measure, don't tune): mcw large-n slice (+0.22% v1,
   registered ~neutral) — if it persists in v2, investigate the auto rule's
   >2000-rows=off assumption on synthetic noise profiles before touching
   anything in the library. Suite runtime: budget the freeze against MEASURED
   all-model cost (CatBoost ≈ 50–70× chimera on synth) so the screen honestly
   lands ≤ 15 min wall, or re-negotiate the budget.

## Then: first real use (the payoff)

The instrument was built to attack the remaining CatBoost classification-Brier
gap (full-suite: their 98.1% vs our 91.4% Brier column). Run cheap screened
sweeps with factor attribution — `leaf_estimation_iterations`, `cat_smoothing`,
calibration-flavored ideas — kill fast, and promote only slice-coherent winners
into `/experiment` (Grinsztajn decides, OpenML gates, TabArena stays sealed).

## Budget

Generator work ~2–3 h; freeze ~20 min; screen baseline ~30 min; 9-arm backtest
~4–5 h (batchable overnight); realism + write-up ~30 min.
