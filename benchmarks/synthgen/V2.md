# SynthGen v2 — executable plan for the next iteration

Self-sufficient handoff: everything needed to build v2 without this file's
authoring session. v1 state: branch `synthgen` (PR #15), gate passed 8/9,
baselines in `benchmarks/results/synv1-{baseline,full-baseline}.json`, corpus
artifact at claude.ai/code/artifact/0326bb6c-6e89-4a56-a224-c7153f2fad3b,
session memory in `project_synthgen.md`.

v2 fixes exactly what the v1 exam indicted. Ground rules: any generator change
⇒ `recipe.VERSION = "v2"` + new `VERSION_SEED` (e.g. 0x53594E32) ⇒ keys become
`syn:v2/*` ⇒ re-freeze + regenerate goldens (v1 golden entries die with v1 —
`parse_key` rejects old versions by design; v1 result JSONs stay readable).
One benchmark at a time. No TabArena in any form. Run script files, never
`python -c`.

## Workstream A — harder targets (fixes the depth4 disagree)

Evidence: depth4 arm +0.31% (should lose); wins concentrated on saturated
cell-rules, n<2000, cat lookups; numeric/n≥2000 slices agree with the ledger.

Changes:
- `recipe.py`: `interaction_depth` → {1:.10, 2:.30, 3:.35, 4:.15, 5:.10};
  `max_in_degree` → 2..5.
- `freeze.py`: stratify screen n to the corpus shape within [600, 8000]
  (cap the n<2000 share at ~35%) instead of greedy small-n fill.
- `backtest.py`: judge depth8/depth4/lr03 arms EXCLUDING saturated sets
  (they reward low capacity by design — same pattern as catcombo's
  deltas_judge filter).

Accept: depth4 flips to loss/not-win; depth8 + lr03 remain losses. If depth4
does not flip, workstream A failed — iterate A before shipping v2 even if the
overall gate passes.

## Workstream B — entity-effect categoricals (fixes the CatBoost-moat gap)

Evidence: realism CHECK — CatBoost>LGBM winrate 0.45 on high-card cats vs
0.62 on numeric (real data shows the opposite; v1 cats are discretized views
of smooth latents).

Changes in `emit.py` (+ a recipe field for the mix):
- New mechanism for ~40% of cat columns: category = latent entity.
  Sample level frequencies Zipf-ish (exponent ~1.2–2), n_levels ≤ 64;
  per-level effect ~ N(0, σ_e), σ_e ~ loguniform(0.3, 1.0). The effect vector
  becomes an SCM node on the target's ancestry (inject as an extra parent of
  a mid node, or a φ contributor) so the column is informative; the OBSERVED
  column is the level label string, never the effect.
- A few test-only rare levels (rows whose label never appears elsewhere) —
  ordered-TS handles unseen levels; that's the realistic stress.
- Keep the discretized-view mechanism for the remaining cat columns.

Accept: the high-card realism CHECK flips to PASS (CatBoost winrate on
card>8 slice > its numeric-slice winrate) AND the backtest still ≥7/9.

## Workstream C — earned canaries (fixes canary-by-construction)

Evidence: v1's cat_cross "saturated" sets were genuinely hard (car-analogs,
+27% under cat_combinations); the screen had zero cat-bearing
verified-at-ceiling canaries (canary slice was empty = vacuously OK).

Changes:
- `freeze.py`: fit default ChimeraBoost on every saturated candidate;
  canary status only if excess Brier ≤ 0.02 (clf) / RMSE ≤ 1.1×σ (reg).
- Canary ids live in `suites.py` as `CANARIES = {...}` next to `SUITES`
  (meta must stay a pure function of (VERSION, id); canary status is
  freeze-time knowledge). `backtest.py`/`synth_report.py` read it from there
  instead of reconstructing from rule_kind.
- Screen must contain ≥3 cat-bearing canaries; keep the hard cat_cross sets
  as the car-analog "+" slice (rule_kind meta already distinguishes).

Accept: catcombo canary slice non-empty and non-positive.

## Watch items (measure, never tune the library from synth)

- mcw large-n slice was +0.22% in v1 (registered ~neutral). Re-read in v2.
- Runtime honesty: v1 "15-min" screen measured ~28 min because CatBoost is
  50–70× ChimeraBoost on synthetic targets (300–700 trees). Default: keep
  ~180-set screens and relabel ~30 min in docs. Alternative if iteration
  speed matters more: freeze to a measured 15-min row budget (~215K rows).

## Order of operations

1. Branch `synthgen-v2` off `synthgen` (or off main post-merge of PR #15).
2. Implement A + B + C; bump VERSION/VERSION_SEED.
3. `python -m pytest tests/test_synthgen.py tests/test_synth_report.py -q`
   (golden test will demand regeneration — expected; regen at step 4).
4. `python benchmarks/synthgen/freeze.py --count 1000` (~20–30 min with the
   new fit checks); paste SUITES + CANARIES + goldens; pytest all green
   (408+ total).
5. Screen baseline:
   `python benchmarks/run_benchmarks.py --synth --seeds 3 --save
   benchmarks/results/synv2-baseline.txt` (~30 min; print the table).
6. `python benchmarks/synthgen/backtest.py --seeds 3 --jobs 5` — overnight
   (~5 h; resumable, skips existing synv2 JSONs after renaming the save
   paths in `_run`/`score` from synv1- to synv2-, a 2-line edit — do it in
   step 2).
7. Gate: ≥7/9 AND canary clean AND the A/B/C acceptance lines above.
8. Realism: `python benchmarks/synth_report.py
   benchmarks/results/synv2-full-baseline.json --realism` after a full-tier
   baseline (~1 h).
9. Docs (this file's verdict section, README, CHANGELOG), commit, push, PR.

## After v2 ships — the payoff program

Target: CatBoost's classification-Brier lead (full suite: 98.1% vs our
91.4% Brier column). Use screened sweeps + factor attribution on
probability-quality levers (`leaf_estimation_iterations`, `cat_smoothing`,
calibration-flavored ideas). Kill fast on the screen; promote slice-coherent
winners into `/experiment` (Grinsztajn decides, OpenML gates, TabArena
sealed). Budget each idea ~1 h screen time vs ~6 h Grinsztajn.

## Budget

(overnight) · full baseline + realism ~1.5 h · docs/PR ~30 min.
